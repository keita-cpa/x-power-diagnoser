# site-optimization.md
## 株式会社MiChi（michi-cpatax.com）コンバージョン最大化 設計図

**最終ゴール:** TallyフォームからのB2B無料相談申し込み ／ Stripeでのデジタルコンテンツ購入
**分析日:** 2026-03-30

---

## 1. 【現状の言語化】

### ターゲット層（ペルソナ）

**メインペルソナ：中小〜中堅企業の経営者・オーナー社長**
- 売上数億〜数百億円規模、従業員50〜500名程度
- M&A、持株会社化、IPO準備などを検討している
- 2026年9月に本格稼働する「KSK2（次世代AI国税システム）」への不安を抱えている
- 現顧問税理士が節税や組織再編に消極的で、物足りなさを感じている
- 役員報酬の手取り最大化、出張旅費規程の適法な導入に興味がある

**サブペルソナ：経理担当者・財務部長**
- 上司（社長）への提案材料として専門コンテンツを探している
- デジタルパッケージ（Excel付き規程）をコスパよく入手したい
- 法的リスクを管理したいが社内に専門家がいない

---

### 独自の強み（バリュープロポジション）

| 強み | 競合との差異化 |
|------|--------------|
| Big4監査法人出身のCPA/税理士 | 一般的な税理士事務所では扱えない組織再編・M&Aに対応 |
| KSK2（AI税務調査）専門対応 | 2026年という時事性・緊急性が高いニーズに直結 |
| Tech × 税務の融合（AI・クラウド） | 「次世代」という差別化軸を持つ |
| 大企業並みの防衛力を中小企業に | 情報格差を解消するという明確なミッション |
| デジタルコンテンツ販売（即日入手） | アドバイザリー以外の低関与接点としてのファネル設計 |

---

### 第一印象の率直な評価

**良い点：**
- デザインはスタイリッシュかつ上品で、"専門家感"が出ている
- KSK2という時流に乗ったテーマで、検索流入が期待できる
- INSIGHTSの記事が充実しており、専門性の証明になっている

**改善が必要な点：**
- ヒーローコピー「複雑な税務課題に、次世代の解決策を」は**抽象的すぎる**。ファーストビューで「誰の・どんな問題を・どう解決するのか」が3秒で伝わらない
- **顔・名前を出さない「ファントム・プロフェッショナル体制」**は、月額数十万〜数百万円のアドバイザリー契約においては、むしろ信頼性の阻害要因になりうる
- サービスの**価格帯が一切不明**なため、自分が対象かどうかの判断ができずに離脱しやすい
- CTAが「お問い合わせ」一本しかなく、**ハードルが高い**。無料相談・資料ダウンロードなどの中間コンバージョンが存在しない

## 2. 【ボトルネックの指摘】

### ボトルネック①：ファーストビューで「誰向けか」が3秒で伝わらない

**現状の問題：**
「複雑な税務課題に、次世代の解決策を。Professional Tax & Tech Advisory by CPA」という英語混じりのコピーは、知的・洗練された印象は与えるが、**経営者が自分ごと化できない**。「あ、これは自分の会社の話だ」と感じさせる具体性がゼロ。

**ユーザー離脱のシナリオ：**
検索でKSK2対策を調べていた社長がランディング → ヒーローを見て「うちに関係あるのかな？」→ スクロールせずに戻る

---

### ボトルネック②：社会的証明（Social Proof）が皆無

**現状の問題：**
- 顧客の声・推薦文：ゼロ
- 導入実績数（匿名でも可）：ゼロ
- メディア掲載・実績数字：ゼロ
- 顔写真・実名公開：ゼロ（ファントム体制）

高額B2Bサービスにおける購買意思決定は「信頼」が最大の阻害要因。専門知識の証明（INSIGHTS記事）はあるが、「本当に依頼して大丈夫か」という**人間的信頼**の裏付けがない。

---

### ボトルネック③：Service IとService IIのターゲット混在によるメッセージの希薄化

**現状の問題：**
「IPO・M&A支援（Service I）」と「規程パッケージ販売（Service II）」はターゲットがまったく異なる。前者は大企業・上場準備企業の意思決定者、後者は中小企業の経営者・経理担当。**1ページに同居させると、どちらのユーザーにも「自分向け」と感じさせにくくなる。**

---

### ボトルネック④：CTAが「お問い合わせ」のみ（ハードルが高すぎる）

**現状の問題：**
ContactページにTallyフォームがあるが、その前の導線に「無料相談」「資料ダウンロード」「まず話を聞く」といった**中間コンバージョンポイント**が存在しない。B2B高額サービスで初回から「お問い合わせ」のみを求めると、検討段階のユーザーは離脱する。

---

### ボトルネック⑤：価格の完全な不透明性

**現状の問題：**
アドバイザリーサービスの費用感が全く記載されていない。デジタルコンテンツの価格もトップページからは確認できない。B2Bでは「問い合わせ後に提案書を送付」という流れは一般的だが、**最低限の価格帯レンジ（例：「月額10万円〜」）を示すことで自己選別が起き、商談の質が向上する**。

---

### ボトルネック⑥：デジタルコンテンツへの導線が弱い

**現状の問題：**
ナビゲーションのドロップダウンにデジタルコンテンツはあるが、トップページのコンテンツフロー上では目立たない。KSK2対策や出張旅費規程の記事を読んでいるユーザーが「今すぐ規程を入手したい」と思っても、購入ページへのCTAが記事内・セクション内に存在しない。

---

## 4. 【実装アクションプラン】

### 前提技術スタック
- **CMS:** WordPress + SWELLテーマ
- **フォーム:** Tally（埋め込み）
- **決済:** Stripe（デジタルコンテンツ購入）
- **ホスティング:** 現状維持（michi-cpatax.com）

---

### STEP 1: Trust Bar（信頼帯）の追加

**使用ブロック:** SWELL「フルワイドブロック」（背景色：ネイビーまたはダーク系）

```
配置場所: ヘッダー直下、ファーストビュー最上部
内容:
  - テキスト「Big4監査法人出身 公認会計士 × 税理士」
  - アイコン付き3カラム: 「KSK2対応」「インボイス登録済」「守秘義務厳守」
実装: SWELLの「カラムブロック（3列）」+ アイコンウィジェット
```

---

### STEP 2: HEROセクションの書き換え

**使用ブロック:** SWELLの「フルワイドブロック」（フルスクリーン高さ設定）

```
実装手順:
1. SWELLエディタ → フルワイドブロック追加
2. 背景: ダーク系グラデーション or 高品質ビジネス系画像
3. 見出しH1: 「2026年9月、AIが税務調査を変える。」
4. 副見出しH2: 「あなたの会社の申告、本当に守れていますか？」
5. 本文（P）: 3行以内の説明コピー
6. CTAボタン（2個）:
   - メイン: 「無料相談を申し込む（30分）」→ #cta-form（同ページのTallyフォームへアンカーリンク）
   - サブ: 「サービス内容を見る」→ #services（同ページ内アンカーリンク）
```

---

### STEP 3: Affinity / Narrow セクション

**使用ブロック:** SWELLの「リッチカラムブロック」+ チェックリスト形式

```
実装手順:
1. SWELLエディタ → リッチカラム（2カラム）
2. 左列: 「こんな不安ありませんか？」（✗マーク付きリスト）
3. 右列: 「こんな方に最適」（✓マーク付きリスト）
4. ブロック背景: ライトグレー（#f5f5f5）で視覚的分離
5. セクション末尾にCTAボタン（中央配置）
```

---

### STEP 4: Offer（サービス2択）セクション

**使用ブロック:** SWELLの「カード型カラムブロック（2列）」

```
実装手順:
1. 左カード: Advisory（相談・顧問）
   - 内容リスト（箇条書き）
   - 価格帯テキスト（例: 「月額¥50,000〜（要相談）」）
   - CTAボタン: 「無料相談を予約する」→ Tally埋め込みフォームへアンカー

2. 右カード: Digital Solutions（デジタルコンテンツ）
   - 商品名リスト（出張旅費規程 / 役員社宅パッケージ）
   - 各価格表示（Stripe連携後の価格）
   - CTAボタン: 「今すぐ購入する」→ Stripeの商品ページへ直リンク

3. SWELLのカードにアクセントカラーの枠・アイコンを追加
```

**Stripe連携:**
```
1. Stripe Dashboardで商品（デジタルコンテンツ）を登録
2. Payment Linkを生成
3. WordPress側にリンクをCTAボタンのhref属性で設定
4. WooCommerceは不要（Payment Linkのみで完結）
5. 購入後の自動納品: Stripeの「Success URL」または
   Tally経由で確認メールにダウンロードURLを記載
```

---

### STEP 5: Social Proof（お客様の声）セクション

**使用ブロック:** SWELLの「口コミ・レビューブロック」または「引用ブロック（カスタム）」

```
実装手順:
1. 匿名の推薦文を2〜3件用意（業種・規模のみ開示）
2. SWELLの「引用ブロック」をカード形式でスタイリング
3. 「守秘義務により匿名表示」の注記を添える
4. 実績数字があれば「導入企業数 ○○社超」などの数値をカウンター表示

⚠️ 重要: 推薦文が用意できない段階では、このセクションは
  「KSK2対策の実績」などの数値コンテンツに一時置換する
```

---

### STEP 6: FAQセクション

**使用ブロック:** SWELLの「アコーディオンブロック（FAQ）」

```
実装手順:
1. SWELLのFAQブロックを追加
2. 既存FAQを上記推奨内容（6項目）に整理
3. 構造化データ（FAQPage schema）をSWELL設定またはプラグインで追加
   → Googleの検索結果でリッチリザルト表示される可能性があり、CTR向上に効果的
4. FAQの最後に「他にご質問がありましたら → 無料相談へ」のテキストリンク
```

---

### STEP 7: Final CTA（Tallyフォーム埋め込み）

**使用ブロック:** SWELLの「フルワイドブロック」（背景色: ダーク系で目立たせる）

```
Tally設定:
1. Tallyダッシュボードで新規フォームを作成
2. 必須フィールド設定:
   - お名前（Short text）
   - 会社名（Short text）
   - 役職（Short text）
   - 年商規模（Dropdown: 〜1億/1〜5億/5〜30億/30億超）
   - ご相談内容（Multiple choice: KSK2/節税/M&A/組織再編/その他）
   - 自由記入欄（Long text、任意）
   - 連絡先メールアドレス（Email）
```
